Description: Verify that browser spell check is active in Docs, Slides text, Slides speaker notes, and Sheets title fields, but not formula inputs.
Prerequisites:
1. Browser or OS spell check is enabled.
2. User is signed in.
3. Docs, Slides, and Sheets editors are available.
Steps:
1. Open a Doc and type helo in the document body.
2. Open a Slide deck and type helo in a text element.
3. Type helo in speaker notes.
4. Open a Sheet and type helo in the sheet title area.
5. Click the formula bar and type =SUM(.
Expected Outcome: Misspelled words receive browser-provided red underlines in document text, slide text, speaker notes, and sheet title. The formula bar does not show spell-check underlines.

Group: Editors
Name: Spell check keyboard toggle persists
Description: Verify that the spell check preference can be toggled and is shared across editors.
Prerequisites:
1. NEXT_PUBLIC_FEATURE_SPELL_CHECK=true.
2. Browser spell check is enabled.
3. User is signed in.
4. A Doc editor has visible misspelled text.
Steps:
1. In the Docs editor, press Cmd+Shift+; on macOS or Ctrl+Shift+; on Windows/Linux.
2. Refresh the page.
3. Open the Slides editor.
4. Press the shortcut again.
5. Refresh both editors.
Expected Outcome: The first shortcut hides spell-check underlines and persists that state in localStorage under neutrino.spellCheck. Slides uses the same preference. The second shortcut re-enables spell check and persists the on state.

Group: Editors
Name: Spell suggestions appear in Docs context menu
Description: Verify that misspelled words show replacement suggestions in the custom Docs context menu.
Prerequisites:
1. NEXT_PUBLIC_FEATURE_SPELL_CHECK_SUGGESTIONS=true.
2. Spell check is enabled.
3. User is signed in.
4. A document is open in /docs/editor.
Steps:
1. Type teh or recieve in the document.
2. Right-click directly on the misspelled word with no text selected.
3. Click one of the suggestions.
4. Right-click a correctly spelled word.
5. Select text and right-click within the selection.
Expected Outcome: The custom context menu opens instead of the browser native menu. Misspelled words show up to five bold suggestions followed by a separator and standard menu items. Choosing a suggestion replaces the word and closes the menu. Correct words and selected text do not show suggestions.

1. Hard-reload the page.
2. Immediately right-click a misspelled word.

Group: Live Sheet Embeds
Name: Paste a copied sheet range into Docs as a live embed
Description: Verify that a copied spreadsheet selection can become a live embed in a document.
Prerequisites:
1. NEXT_PUBLIC_FEATURE_SHEET_LIVE_EMBED=true if the feature is gated.
2. Sheets backend supports named ranges.
3. User is signed in.
4. A spreadsheet with data exists.
5. A document is open in /docs/editor.
Steps:
1. Open the spreadsheet.
2. Select a range with visible values.
3. Copy the selection.
4. Switch to the document.
5. Paste at the desired insertion point.
6. Choose the live embed option if prompted.
Expected Outcome: A Live sheet embed block appears in the document and displays the copied table data with the expected header controls.

Group: Live Sheet Embeds
Name: Refresh a live sheet embed after source data changes
Description: Verify that a live embed can fetch updated source sheet data.
Prerequisites:
1. A document contains a live sheet embed.
2. The source spreadsheet still exists.
3. User is signed in.
Steps:
1. Open the source spreadsheet.
2. Change one or more cells in the embedded range.
3. Return to the document.
4. Click Check for updates on the embed.
Expected Outcome: The embed refreshes to show the updated cell values and retains its document placement.
